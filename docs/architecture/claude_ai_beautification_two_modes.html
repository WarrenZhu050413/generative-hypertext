AI Beautification: Two-Mode System

✨ The Corrected Vision

Current flow remains unchanged: Capture HTML → Sanitize with DOMPurify → Save original HTML + screenshot → Display in Canvas (may look broken)

New feature: Two AI beautification modes that users can trigger on-demand

🎨 Mode 1: Recreate Design

Claude analyzes screenshot + original HTML and tries to recreate the visual appearance as faithfully as possible

Use case: Visual fidelity matters

📋 Mode 2: Organize Information

Claude reads original HTML only and extracts/organizes information in a clean, readable format

Use case: Content matters more than design

Complete Data Flow

▶ Step-by-Step Flow
1. Capture (Current System)
User presses Cmd+Shift+E → ElementSelector activates → User clicks element
2. Process Capture (Current System)
• Extract HTML from element
• Sanitize with DOMPurify (remove scripts, iframes, etc.)
• Capture screenshot (optional, non-blocking)
• Compress screenshot
3. Store Data (Current System)
• Card metadata + sanitized HTML → chrome.storage.local
• Screenshot → IndexedDB
• NEW: Also store ORIGINAL unsanitized HTML for AI processing
4. Display in Canvas (Current System)
Card shows sanitized HTML (may look broken/unstyled) + screenshot fallback
5. User Chooses Beautification Mode (NEW)
User clicks one of two buttons:
• "🎨 Recreate Design" → Mode 1
• "📋 Organize Content" → Mode 2
6. AI Processing (NEW)
Mode 1: Send screenshot (vision) + original HTML → Claude recreates design
Mode 2: Send original HTML only → Claude organizes information
7. Re-sanitize & Store (NEW)
• Re-sanitize LLM output with DOMPurify
• Save as card.beautifiedContent
• Track which mode was used
8. Display Options (NEW)
User can toggle between:
• Original (sanitized HTML)
• Beautified (AI-generated HTML)
• Screenshot

Mode Comparison

🎨 Mode 1: Recreate Design

Input to Claude:

  • Screenshot (via vision API)
  • Original unsanitized HTML
  • Metadata (URL, title, domain)

Prompt Goal:

"Look at this screenshot and HTML. Recreate the visual design as faithfully as possible using self-contained HTML with inline styles."

Best For:

  • Infographics
  • Dashboards
  • Design examples
  • Data visualizations
  • Styled UI components
  • Complex layouts

Example Use Cases:

  • Capturing a GitHub repo stats card
  • Saving a Twitter poll visualization
  • Clipping a product comparison table
  • Preserving a styled code example

Output Characteristics:

  • Matches original colors, fonts, spacing
  • Preserves layout (grid, flex, positioning)
  • Similar visual hierarchy
  • May be less readable but more authentic

📋 Mode 2: Organize Information

Input to Claude:

  • Original unsanitized HTML
  • Metadata (URL, title, domain)
  • NO screenshot

Prompt Goal:

"Extract the meaningful information from this HTML and present it in a clean, readable format. Ignore the original design."

Best For:

  • Articles & blog posts
  • Documentation
  • Tutorial content
  • Reference material
  • Text-heavy pages
  • Long-form content

Example Use Cases:

  • Saving a Medium article (remove ads/nav)
  • Clipping Stack Overflow answers
  • Capturing API documentation
  • Archiving research papers (HTML version)

Output Characteristics:

  • Clean typography (Chinese aesthetic)
  • Semantic organization (headings, lists, code)
  • Removes clutter (ads, navs, sidebars)
  • Optimized for readability
  • Consistent styling across all cards

  // Extract raw HTML (unsanitized)
  const originalHTML = element.outerHTML;  // NEW: Keep original

  const sanitizedHTML = DOMPurify.sanitize(originalHTML, {

  // Capture screenshot (existing)
  let screenshotId: string | undefined;
  try {
    const screenshot = await captureElementScreenshot(element);

  }

  // Create card

    id: generateId(),
    content: sanitizedHTML,        // Sanitized for display
    originalHTML: originalHTML,    // NEW: Unsanitized for AI
    metadata: {
      url: window.location.href,
      title: document.title,
      domain: window.location.hostname,
      // ...
    },
    screenshotId,
    starred: false,
    tags: [],
    createdAt: Date.now(),
    updatedAt: Date.now(),
  };

  await saveCard(card);
};
Security Note: originalHTML is stored but NEVER rendered directly. It's only used as input to the AI, and the AI's output is re-sanitized before display.

▶ Mode 1: Recreate Design Implementation
src/utils/beautification.ts
import Anthropic from '@anthropic-ai/sdk';
import DOMPurify from 'dompurify';

export async function recreateDesign(
  card: Card,
  screenshotBase64: string
): Promise<string> {
  const client = new Anthropic({
    apiKey: await getApiKey(),
  });

  const systemPrompt = `You are an expert web designer.
You will receive a screenshot and the original HTML of a web element.

TASK: Recreate the visual design as faithfully as possible using
self-contained HTML with inline styles.

REQUIREMENTS:
1. Match colors, fonts, spacing, layout from the screenshot
2. Use inline <style> tag - NO external CSS
3. Preserve all content from the HTML
4. Make it pixel-perfect if possible
5. Use flexbox/grid to match layout
6. Output ONLY the HTML (no explanations)

OUTPUT FORMAT:
<style>
  /* Your CSS here */
</style>
<div class="recreated-content">
  <!-- HTML here -->
</div>`;

  const response = await client.messages.create({
    model: 'claude-sonnet-4',
    max_tokens: 4096,
    messages: [{
      role: 'user',
      content: [
        {
          type: 'image',
          source: {
            type: 'base64',
            media_type: 'image/png',
            data: screenshotBase64,
          },
        },
        {
          type: 'text',
          text: `Here is the original HTML:\n\n${card.originalHTML}\n\nRecreate this design as faithfully as possible.`,
        },
      ],
    }],
    system: systemPrompt,
  });

  const beautifiedHTML = response.content[0].text;

  // CRITICAL: Re-sanitize AI output
  return DOMPurify.sanitize(beautifiedHTML, {
    ALLOWED_TAGS: [
      'div', 'span', 'p', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6',
      'ul', 'ol', 'li', 'a', 'img', 'code', 'pre', 'blockquote',
      'table', 'thead', 'tbody', 'tr', 'th', 'td', 'style',
      'br', 'strong', 'em', 'section', 'article', 'header', 'footer'
    ],
    ALLOWED_ATTR: ['style', 'class', 'id', 'href', 'src', 'alt'],
    ALLOW_DATA_ATTR: false,
  });
}

export async function organizeInformation(
  card: Card
): Promise<string> {
  const client = new Anthropic({
    apiKey: await getApiKey(),
  });

  const systemPrompt = `You are an expert content organizer.
You will receive raw HTML from a captured web element.

TASK: Extract meaningful information and present it in a clean,
readable format optimized for human consumption.

REQUIREMENTS:
1. Extract ONLY the main content (ignore ads, navigation, sidebars)
2. Organize with semantic structure (headings, lists, code blocks)
3. Use Chinese aesthetic styling:
   - Red (#8B0000) for headings and accents
   - Gold (#FFD700) for highlights
   - Beige (#F5F5DC) gradient background
   - Clean, elegant, minimal
4. Apply proper typography hierarchy
5. Format code blocks with syntax-appropriate styling
6. Remove clutter but preserve all meaningful information
7. Use inline <style> tag - NO external CSS
8. Output ONLY the HTML (no explanations)

OUTPUT FORMAT:
<style>
  /* Beautiful Chinese aesthetic CSS */
</style>
<div class="organized-content">
  <!-- Clean, organized HTML -->
</div>`;

  const response = await client.messages.create({
    model: 'claude-sonnet-4',
    max_tokens: 4096,
    messages: [{
      role: 'user',
      content: `Here is the HTML to organize:\n\n<metadata>\nURL: ${card.metadata.url}\nTitle: ${card.metadata.title}\n</metadata>\n\n<html>\n${card.originalHTML}\n</html>\n\nExtract and organize this information in a beautiful, readable format.`,
    }],
    system: systemPrompt,
  });

  const beautifiedHTML = response.content[0].text;

  // Re-sanitize
  return DOMPurify.sanitize(beautifiedHTML, {
    ALLOWED_TAGS: [
      'div', 'span', 'p', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6',
      'ul', 'ol', 'li', 'a', 'img', 'code', 'pre', 'blockquote',
      'table', 'thead', 'tbody', 'tr', 'th', 'td', 'style',
      'br', 'strong', 'em', 'section', 'article'
    ],
    ALLOWED_ATTR: ['style', 'class', 'id', 'href', 'src', 'alt'],
    ALLOW_DATA_ATTR: false,
  });
}

src/canvas/CardNode.tsx
const CardNode = ({ data }: { data: Card }) => {
  const [isProcessing, setIsProcessing] = useState(false);
  const [displayMode, setDisplayMode] = useState<Card['displayMode']>(
    data.displayMode || 'original'
  );

  const handleRecreateDesign = async () => {
    if (!data.screenshotId) {
      alert('Screenshot required for design recreation');
      return;
    }

    setIsProcessing(true);
    try {
      // Load screenshot from IndexedDB
      const screenshot = await loadScreenshotFromIndexedDB(data.screenshotId);
      const base64 = await blobToBase64(screenshot);

      // Call Mode 1
      const beautified = await recreateDesign(data, base64);

      // Update card
      const updated: Card = {
        ...data,
        beautifiedContent: beautified,
        beautificationMode: 'recreate',
        beautificationDate: Date.now(),
        displayMode: 'beautified',
      };

      await updateCard(updated);
      setDisplayMode('beautified');
    } catch (error) {
      console.error('[CardNode] Recreation failed:', error);
      alert('Failed to recreate design: ' + error.message);
    } finally {
      setIsProcessing(false);
    }
  };

  const handleOrganizeContent = async () => {
    setIsProcessing(true);
    try {
      // Call Mode 2 (no screenshot needed)
      const beautified = await organizeInformation(data);

      // Update card
      const updated: Card = {
        ...data,
        beautifiedContent: beautified,
        beautificationMode: 'organize',
        beautificationDate: Date.now(),
        displayMode: 'beautified',
      };

      await updateCard(updated);
      setDisplayMode('beautified');
    } catch (error) {
      console.error('[CardNode] Organization failed:', error);
      alert('Failed to organize content: ' + error.message);
    } finally {
      setIsProcessing(false);
    }
  };

  const renderContent = () => {
    if (displayMode === 'screenshot' && data.screenshotId) {
      return <ScreenshotView screenshotId={data.screenshotId} />;
    }
    if (displayMode === 'beautified' && data.beautifiedContent) {
      return (
        <div
          className="beautified-content"
          dangerouslySetInnerHTML={{ __html: data.beautifiedContent }}
        />
      );
    }
    // Default: original sanitized content
    return (
      <div
        className="original-content"
        dangerouslySetInnerHTML={{ __html: data.content }}
      />
    );
  };

  return (
    <div className="card-node">
      {/* View tabs */}
      <div className="view-tabs">
        <button onClick={() => setDisplayMode('original')}>
          Original
        </button>
        {data.beautifiedContent && (
          <button onClick={() => setDisplayMode('beautified')}>
            ✨ {data.beautificationMode === 'recreate' ? 'Recreated' : 'Organized'}
          </button>
        )}
        {data.screenshotId && (
          <button onClick={() => setDisplayMode('screenshot')}>
            Screenshot
          </button>
        )}
      </div>

      {/* Content */}
      {renderContent()}

      {/* Beautification actions */}
      {!data.beautifiedContent && (
        <div className="beautify-actions">
          <button
            onClick={handleRecreateDesign}
            disabled={isProcessing || !data.screenshotId}
            title={!data.screenshotId ? 'Screenshot required' : ''}
          >
            {isProcessing ? '⏳ Processing...' : '🎨 Recreate Design'}
          </button>
          <button
            onClick={handleOrganizeContent}
            disabled={isProcessing}
          >
            {isProcessing ? '⏳ Processing...' : '📋 Organize Content'}
          </button>
        </div>
      )}

      {/* Regenerate option */}
      {data.beautifiedContent && (
        <div className="beautify-actions">
          <button onClick={
            data.beautificationMode === 'recreate'
              ? handleRecreateDesign
              : handleOrganizeContent
          }>
            🔄 Regenerate
          </button>
        </div>
      )}
    </div>
  );
};

Key Advantages of This Approach

🎯 Why This Design Is Superior

✅ User Control

  • Cards display quickly (no AI processing on capture)
  • Users choose IF and HOW to beautify
  • Can keep original + beautified versions

✅ Flexibility

  • Two modes for different use cases
  • Screenshot-based OR content-based
  • Can regenerate with different modes

✅ Performance

  • Fast capture (current system unchanged)
  • AI processing only when requested
  • No API costs for users who don't want it

✅ Security

  • Original HTML never rendered directly
  • AI output always re-sanitized
  • User's API key (not embedded in extension)
