Option A: Manual Migration (Recommended)

mkdir -p claude_html
mv claude_*.html claude_html/
cd claude_html
for file in claude_*.html; do mv "$file" "${file#claude_}"; done

Option B: Automated Migration Script

Create a bash script for automated migration:

#!/bin/bash
# migrate_claude_html.sh
# Migrates claude_*.html files to claude_html/ directory

mkdir -p claude_html
for file in claude_*.html; do
    if [ -f "$file" ]; then
        newname="${file#claude_}"
        mv "$file" "claude_html/$newname"
        echo "Moved: $file ‚Üí claude_html/$newname"
    fi
done
echo "Migration complete!"

Option C: Leave in Place

  • Keep existing files where they are
  • Only new files use new system
  • Gradually phase out old files over time
  • Pros: No risk of data loss, simple
  • Cons: Inconsistent file locations during transition
