10:42 AM

Analysis of information seeking in Information Seeking in Electronic Environments shows information seeking is highly non-linear environment, involving many subprocesses that may go to each other.

PARALLEL SUBPROCESS ACTIVATION:

1. Continuous Background Processes (Sentinels) - Define Problem remains active throughout search, Recognize Problem stays alert for related needs, Reflect continuously monitors progress

2. Opportunistic Branching - While extracting from one source, execute search in another tab; While examining results, simultaneously formulate refined query; Select alternative systems while waiting for execution

3. Cognitive Parallelism - Pattern recognition while actively reading, associative thinking across tasks, subconscious incubation of problems

4. Interleaved Processing - Rapid switching that appears parallel: Examine + Define + Reflect in quick succession

KEY INSIGHTS:

  • Figure 3.3 shows 8 subprocesses with three transition types: Default (most likely), High Probability (dashed), Low Probability (thin)
  • Define Problem and Examine Results are critical hub nodes - most processes return to these
  • Electronic environments enable more parallelism through multiple windows/tabs, instant execution, copy/paste between sources
  • Experts can maintain more parallel processes due to automated skills and pattern recognition
  • Browsing has more parallel activations than analytical search
  • Interruptions are costly because they disrupt multiple parallel processes simultaneously

THEORETICAL IMPLICATIONS:

  • Information seeking is not pipeline but complex system with dynamic interactions
  • Information seeker acts as conductor orchestrating parallel activities based on goals, opportunities, constraints
  • Electronic systems can overwhelm cognitive capacity by enabling more parallelism than we can handle
  • Model distinguishes from simpler linear search models by showing recursive, parallel, opportunistic subprocess activation

Building Nabokov's Cards system with spatial arrangement and modular export

KEY INSIGHTS FROM PAPER:

  • Bottom-up creativity: fragments combine into larger ideas
  • Non-linear workflows essential for creative writing
  • Drag-and-drop metaphor encourages experimentation
  • Modular thinking helps writers see patterns

IMPLEMENTATION PLAN:

1. Spatial Canvas

  • Infinite scrollable workspace
  • Cards positioned absolutely with (x,y) coords
  • Proximity = implicit relationships
  • Visual clustering for thematic grouping

2. Card as Atomic Unit

  • Each card fully self-contained
  • Multiple export formats (JSON/MD/HTML)
  • Unique IDs for tracking/combination
  • Metadata for context (tags, timestamps, connections)

3. Combination Mechanics

  • Drag card onto another to combine (like paper)
  • LLM generates fusion based on content types
  • Decouple to restore originals
  • Regenerate for variations

4. Export Architecture

  • Every HTML element has data-export attribute
  • Right-click any element â†’ Export as...
  • Batch export selected cards
  • Maintain relationships in export

5. Colocation Principle

  • Related cards physically near each other
  • Distance = conceptual distance
  • Stash areas for temporary storage
  • Multiple workspaces for different projects

TECHNICAL STACK:

  • React with react-beautiful-dnd
  • Canvas/SVG for connection visualization
  • IndexedDB for persistence
  • GPT-4 for combinations
  • Electron for desktop app (optional)
