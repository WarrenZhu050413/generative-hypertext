Side Panel Architecture - Card Display Redesign

Key Architecture Changes

  • Inline Image Display: Image cards now render card.imageData directly in the card body
  • Canvas-like Styling: Cards use same gradient background, border radius, and shadow as CardNode
  • Improved Layout: Header, content, footer sections match canvas card structure
  • Card Type Badges: Visual indicators for image/note/clipped cards
  • Responsive Images: Max height of 300px with contain object-fit to preserve aspect ratio

TypeScript: Conditional Rendering Logic

// In SidePanel.tsx - Card rendering logic

{filteredCards.map((card) => {
  return (
    <div key={card.id} css={cardItemStyles}>
      {/* Header */}
      <div css={cardHeaderStyles}>
        <div css={cardHeaderLeftStyles}>
          {card.metadata.favicon && <span>{card.metadata.favicon}</span>}
          <span>{card.metadata.domain}</span>
          {/* Card Type Badge */}
          <span css={cardTypeBadgeStyles(card.cardType)}>
            {card.cardType}
          </span>
        </div>
        <span>{formatDate(card.createdAt)}</span>
      </div>

      {/* Title */}
      <div css={cardTitleStyles}>{card.metadata.title}</div>

      {/* CRITICAL: Conditional Image Display */}
      {card.cardType === 'image' && card.imageData ? (
        <div css={imageContainerStyles}>
          <img
            src={card.imageData}
            alt={card.metadata.title}
            css={imageStyles}
          />
        </div>
      ) : (
        /* Text Content */
        card.content && (
          <div css={contentStyles}>
            <div dangerouslySetInnerHTML={{ __html: sanitizedContent }} />
          </div>
        )
      )}

      {/* Tags */}
      {card.tags?.length > 0 && (
        <div css={tagsStyles}>
          {card.tags.map((tag, i) => (
            <span key={i} css={tagStyles}>{tag}</span>
          ))}
        </div>
      )}

      {/* Actions */}
      <div css={actionsStyles}>
        <button onClick={() => handleRestore(card)}>
          ↩️ Restore
        </button>
        <button onClick={() => handleDelete(card)}>
          🗑️ Delete
        </button>
      </div>
    </div>
  );
})}

Key Differences: Before vs After

Before (Current)
  • Simple card list with flat styling
  • No image display for image cards
  • Only shows text content preview
  • Different visual style from canvas
  • No card type indicators
After (New Design)
  • Canvas-like card styling with gradients
  • Inline image display for image cards
  • Conditional rendering based on cardType
  • Visual consistency with canvas cards
  • Card type badges for quick identification
