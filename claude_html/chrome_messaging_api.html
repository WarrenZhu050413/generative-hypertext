üîå Chrome Messaging API - Complete Guide

‚ö° Core Concept: Message Passing Between Isolated Contexts

Chrome extensions have multiple isolated execution contexts (background, content scripts, popups, devtools). They cannot share memory or variables. The Messaging API enables communication between these contexts using:

  • One-time messages - Simple request/response (sendMessage)
  • Long-lived connections - Persistent channels (connect)
  • External messaging - Communicate with other extensions/web pages

üéØ Three Types of Messaging

1Ô∏è‚É£ One-Time Messages

Use case: Simple request/response (like HTTP)

Methods:

  • chrome.runtime.sendMessage()
  • chrome.tabs.sendMessage()

Best for: Single queries, commands, notifications

2Ô∏è‚É£ Long-Lived Connections

Use case: Persistent communication channel

Methods:

  • chrome.runtime.connect()
  • chrome.tabs.connect()

Best for: Streams, real-time updates, multiple exchanges

3Ô∏è‚É£ External Messaging

Use case: Communication with other extensions or web pages

Methods:

  • chrome.runtime.sendMessage(extensionId, message) - To other extension
  • window.postMessage() - Between content script and web page

Best for: Extension interoperability, web page interaction

üîÑ Pattern 1: One-Time Messages (Most Common)

Sequence Flow

üì§ Sending (Content Script ‚Üí Background)

// Content script sends message
chrome.runtime.sendMessage(
  {
    action: 'CAPTURE_ELEMENT',
    data: {
      html: elementHTML,
      url: window.location.href
    }
  },
  (response) => {
    // Handle response
    if (response.success) {
      console.log('Captured:', response.cardId);
    }
  }
);

üì• Receiving (Background Worker)

// Background worker receives
chrome.runtime.onMessage.addListener(
  (message, sender, sendResponse) => {
    if (message.action === 'CAPTURE_ELEMENT') {
      // Process the capture
      const cardId = saveCard(message.data);

      // Send response back
      sendResponse({
        success: true,
        cardId: cardId
      });

      return true; // CRITICAL!
    }
  }
);
‚ö†Ô∏è CRITICAL: Must return true in listener if sending async response. Otherwise channel closes immediately and sendResponse won't work!
üîç Advanced: Async/Await Pattern for sendMessage ‚ñ∂

Wrap sendMessage in Promise for cleaner async code:

// Utility function
function sendMessageAsync(message) {
  return new Promise((resolve, reject) => {
    chrome.runtime.sendMessage(message, (response) => {
      if (chrome.runtime.lastError) {
        reject(chrome.runtime.lastError);
      } else {
        resolve(response);
      }
    });
  });
}

// Usage with async/await
async function captureElement() {
  try {
    const response = await sendMessageAsync({
      action: 'CAPTURE_ELEMENT',
      data: elementData
    });
    console.log('Success:', response.cardId);
  } catch (error) {
    console.error('Failed:', error);
  }
}

üîó Pattern 2: Long-Lived Connections

Sequence Flow

sequenceDiagram participant CS as Content Script participant BG as Background Worker CS->>BG: connect('channel-name') Note over CS,BG: Connection established loop Multiple messages CS->>BG: port.postMessage({type: 'UPDATE'}) BG->>CS: port.postMessage({type: 'ACK'}) end Note over CS: Done communicating CS->>BG: port.disconnect() Note over CS,BG: Connection closed

üì§ Initiating Connection

// Content script creates connection
const port = chrome.runtime.connect({
  name: 'monitoring-channel'
});

// Send messages on port
port.postMessage({
  type: 'STATUS_UPDATE',
  status: 'processing'
});

// Listen for responses
port.onMessage.addListener((msg) => {
  console.log('Received:', msg);
});

// Handle disconnection
port.onDisconnect.addListener(() => {
  console.log('Disconnected');
});

// Later: close connection
port.disconnect();

üì• Accepting Connection

// Background worker accepts
chrome.runtime.onConnect.addListener(
  (port) => {
    console.log('Connected:', port.name);

    // Listen to messages on this port
    port.onMessage.addListener((msg) => {
      if (msg.type === 'STATUS_UPDATE') {
        // Send message back
        port.postMessage({
          type: 'ACK',
          received: true
        });
      }
    });

    port.onDisconnect.addListener(() => {
      console.log('Port disconnected');
    });
  }
);
üí° When to use connections vs one-time messages:
  • Use one-time messages for: Single requests, commands, simple data fetch
  • Use connections for: Streaming data, multiple back-and-forth exchanges, real-time updates, WebSocket-like behavior

üì® Sending Messages to Content Scripts

Background workers use chrome.tabs.sendMessage() to send to content scripts. Must specify target tab ID.

sequenceDiagram participant BG as Background Worker participant Tab as chrome.tabs API participant CS as Content Script (Tab 123) Note over BG: Need to message specific tab BG->>Tab: tabs.query({active: true}) Tab-->>BG: [{id: 123, ...}] BG->>CS: tabs.sendMessage(123, {action: 'HIGHLIGHT'}) CS-->>BG: sendResponse({done: true}) 
// Background worker sends to content script
async function messageActiveTab() {
  // Find active tab
  const [tab] = await chrome.tabs.query({
    active: true,
    currentWindow: true
  });

  // Send message to that tab's content script
  const response = await chrome.tabs.sendMessage(tab.id, {
    action: 'HIGHLIGHT_ELEMENT',
    selector: '.important'
  });

  console.log('Content script responded:', response);
}

// Content script receives
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  if (message.action === 'HIGHLIGHT_ELEMENT') {
    document.querySelector(message.selector)?.classList.add('highlight');
    sendResponse({done: true});
  }
});

üåê Pattern 3: External Messaging

A) Content Script ‚Üî Web Page

‚ö†Ô∏è Important: Content scripts and web pages have separate JavaScript environments. They share the DOM but not variables. Use window.postMessage() to communicate.
sequenceDiagram participant WEB as Web Page JS participant DOM as Shared DOM participant CS as Content Script Note over WEB: User action in page WEB->>DOM: window.postMessage({type: 'REQUEST'}, '*') DOM->>CS: message event Note over CS: Process request CS->>DOM: window.postMessage({type: 'RESPONSE'}, '*') DOM->>WEB: message event

Web Page Sends

// Web page JavaScript
window.postMessage({
  type: 'FROM_PAGE',
  action: 'getData'
}, '*');

// Listen for response
window.addEventListener('message', (e) => {
  if (e.source === window &&
      e.data.type === 'FROM_EXTENSION') {
    console.log('Got:', e.data);
  }
});

Content Script Receives

// Content script
window.addEventListener('message', (e) => {
  if (e.source === window &&
      e.data.type === 'FROM_PAGE') {

    // Process and respond
    window.postMessage({
      type: 'FROM_EXTENSION',
      result: 'processed'
    }, '*');
  }
});

B) Extension ‚Üî Other Extension

// Your extension sends to another extension
chrome.runtime.sendMessage(
  'other-extension-id-here',
  {action: 'getInfo'},
  (response) => {
    console.log('Other extension responded:', response);
  }
);

// Other extension must declare in manifest.json:
// "externally_connectable": {
//   "ids": ["your-extension-id"]
// }

async function captureElement(element: HTMLElement) {
  // Extract HTML
  const html = element.outerHTML;

  // Request screenshot from background
  const response = await chrome.runtime.sendMessage({
    action: 'CAPTURE_SCREENSHOT'
  });

  // Save card
  const card = {
    id: generateId(),
    content: DOMPurify.sanitize(html),
    screenshotId: response.screenshot ? generateId() : undefined,
    metadata: {
      url: window.location.href,
      title: document.title
    }
  };

  // Save to storage
  const {cards = []} = await chrome.storage.local.get('cards');
  await chrome.storage.local.set({cards: [...cards, card]});

  // Dispatch event for canvas to refresh
  window.dispatchEvent(new CustomEvent('nabokov:cards-updated'));
}

‚ö†Ô∏è Common Pitfalls & Solutions

‚ùå Pitfall 1: "Could not establish connection. Receiving end does not exist" ‚ñ∂

‚ùå Problem

Sending message to context that doesn't exist or hasn't registered listener yet

// Content script not loaded yet
chrome.tabs.sendMessage(tabId, message);
// Error: Receiving end does not exist

‚úÖ Solution

Check for errors and ensure target is ready

chrome.tabs.sendMessage(tabId, msg, (resp) => {
  if (chrome.runtime.lastError) {
    console.warn('Target not ready:',
      chrome.runtime.lastError.message);
    return;
  }
  // Handle response
});
‚ùå Pitfall 2: sendResponse doesn't work ‚ñ∂

‚ùå Problem

Forgot to return true for async response

chrome.runtime.onMessage.addListener(
  (msg, sender, sendResponse) => {
    setTimeout(() => {
      sendResponse({data: 'hello'});
    }, 1000);
    // Missing return true!
  }
);

‚úÖ Solution

Always return true if responding async

chrome.runtime.onMessage.addListener(
  (msg, sender, sendResponse) => {
    setTimeout(() => {
      sendResponse({data: 'hello'});
    }, 1000);
    return true; // Keep channel open!
  }
);
‚ùå Pitfall 3: Multiple listeners responding ‚ñ∂

‚ùå Problem

Multiple listeners all return true, but only first response is used

// Listener 1
chrome.runtime.onMessage.addListener(...);
// Listener 2 (also handles same message!)
chrome.runtime.onMessage.addListener(...);
// Only first response received!

‚úÖ Solution

Use switch statement or return false if not handling

chrome.runtime.onMessage.addListener(
  (msg, sender, sendResponse) => {
    switch(msg.action) {
      case 'MY_ACTION':
        handleMyAction();
        return true;
      default:
        return false; // Don't handle
    }
  }
);
‚ùå Pitfall 4: Port disconnects unexpectedly ‚ñ∂

‚ùå Problem

Port closes when page unloads or extension context invalidated

port.postMessage({data: 'hello'});
// Error: Attempting to use
// disconnected port

‚úÖ Solution

Always handle onDisconnect and reconnect if needed

port.onDisconnect.addListener(() => {
  console.log('Port closed:',
    chrome.runtime.lastError);
  // Reconnect if needed
  port = chrome.runtime.connect({
    name: 'my-channel'
  });
});

üîê Security Considerations

1. Validate Message Sources

chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => {
  // Verify sender is from your extension
  if (sender.id !== chrome.runtime.id) {
    console.warn('Message from unknown extension:', sender.id);
    return;
  }

  // For content scripts, verify URL
  if (sender.tab && !sender.tab.url.startsWith('https://trusted-domain.com')) {
    console.warn('Message from untrusted page:', sender.tab.url);
    return;
  }

  // Process message
});

2. Sanitize All Data

chrome.runtime.onMessage.addListener((msg) => {
  if (msg.action === 'SAVE_HTML') {
    // NEVER trust raw HTML from content scripts
    const sanitized = DOMPurify.sanitize(msg.html);
    saveToStorage(sanitized);
  }
});

3. Use Specific Origins for postMessage

// ‚ùå BAD - accepts from any origin
window.postMessage(data, '*');

// ‚úÖ GOOD - specific origin
window.postMessage(data, 'https://trusted-site.com');

üí° Best Practices Summary

  • ‚úÖ Always check chrome.runtime.lastError in callbacks
  • ‚úÖ Return true from onMessage listeners if responding asynchronously
  • ‚úÖ Use sendMessage for simple request/response patterns
  • ‚úÖ Use connect for long-lived connections or streaming data
  • ‚úÖ Validate sender identity in all message listeners
  • ‚úÖ Handle disconnection gracefully with onDisconnect listeners
  • ‚úÖ Sanitize all data received from content scripts or web pages
  • ‚úÖ Use storage events (chrome.storage.onChanged) for state sync across contexts
  • ‚ùå Don't send functions, DOM nodes, or complex objects - messages are JSON-serialized
  • ‚ùå Don't rely on global variables - contexts are isolated
