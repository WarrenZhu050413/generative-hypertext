Method 2: Custom Component

import { Prism as SyntaxHighlighter } from
  'react-syntax-highlighter'
import { dark } from
  'react-syntax-highlighter/dist/esm/styles/prism'

<Markdown
  components={{
    code({node, inline, className, children}) {
      const match = /language-(\w+)/.exec(className || '')
      return !inline && match ? (
        <SyntaxHighlighter
          style={dark}
          language={match[1]}
          PreTag="div"
        >
          {String(children).replace(/\n$/, '')}
        </SyntaxHighlighter>
      ) : (
        <code className={className}>
          {children}
        </code>
      )
    }
  }}
>
  {markdown}
</Markdown>

‚ñ∂Common Patterns

Loading from File

import { useEffect, useState } from 'react'
import Markdown from 'react-markdown'

function MarkdownPage() {
  const [content, setContent] = useState('')

  useEffect(() => {
    fetch('/content.md')
      .then(res => res.text())
      .then(text => setContent(text))
  }, [])

  return <Markdown>{content}</Markdown>
}

Dynamic Routing

import { useParams } from 'react-router-dom'

function MarkdownPage() {
  const { filename } = useParams()
  const [content, setContent] = useState('')

  useEffect(() => {
    fetch(`/posts/${filename}.md`)
      .then(res => res.text())
      .then(text => setContent(text))
  }, [filename])

  return <Markdown>{content}</Markdown>
}

Live Preview Editor

import { useState } from 'react'
import Markdown from 'react-markdown'

function Editor() {
  const [markdown, setMarkdown] = useState('# Hello')

  return (
    <div style={{display: 'grid', gridTemplateColumns: '1fr 1fr'}}>
      <textarea
        value={markdown}
        onChange={(e) => setMarkdown(e.target.value)}
      />
      <Markdown>{markdown}</Markdown>
    </div>
  )
}
‚ñ∂Security Best Practices
CRITICAL

Built-in Security

‚úÖ react-markdown is safe by default - no dangerouslySetInnerHTML or XSS vulnerabilities

URL Sanitization

import { defaultUrlTransform } from 'react-markdown'

// Custom URL transform
const safeUrlTransform = (url) => {
  // Only allow https URLs
  if (url.startsWith('https://')) {
    return url
  }
  return ''
}

<Markdown urlTransform={safeUrlTransform}>
  {markdown}
</Markdown>

Element Filtering

// Whitelist approach (recommended)
<Markdown allowedElements={['p', 'strong', 'em', 'a']}>
  {userContent}
</Markdown>

// Blacklist approach
<Markdown disallowedElements={['script', 'iframe']}>
  {userContent}
</Markdown>

HTML Handling

// Skip HTML entirely (safest)
<Markdown skipHtml>{content}</Markdown>

// Allow HTML but sanitize (use rehype-sanitize)
import rehypeSanitize from 'rehype-sanitize'

<Markdown rehypePlugins={[rehypeSanitize]}>
  {content}
</Markdown>
‚ö†Ô∏è User Content: When rendering user-generated content, always use skipHtml or rehypeSanitize

Debounced Updates

import { useState, useCallback } from 'react'
import debounce from 'lodash.debounce'

function Editor() {
  const [content, setContent] = useState('')

  const handleChange = useCallback(
    debounce((value) => setContent(value), 300),
    []
  )

  return (
    <>
      <textarea onChange={(e) => handleChange(e.target.value)} />
      <Markdown>{content}</Markdown>
    </>
  )
}

Processing Pipeline

    ‚Üì

    ‚Üì
remark plugins ‚Üí Modified MDAST
    ‚Üì
remark-rehype (converter) ‚Üí HAST (HTML AST)
    ‚Üì
rehype plugins ‚Üí Modified HAST
    ‚Üì
React component mapping ‚Üí React Elements
    ‚Üì
Rendered Output

Key Libraries Used

  • remark: Markdown parser and processor
  • unified: Interface for processing text with syntax trees
  • rehype: HTML processor
  • mdast: Markdown Abstract Syntax Tree specification
  • hast: HTML Abstract Syntax Tree specification

AST Node Types

MDAST (Markdown)

  • root, paragraph, heading
  • text, emphasis, strong
  • link, image, list, listItem
  • code, inlineCode, blockquote

HAST (HTML)

  • element, text, root
  • comment, doctype
  • Properties (className, href, etc.)
  • Children array

‚ñ∂Common Issues & Solutions

Issue: "Cannot use import statement"

Cause: ES module compatibility 
// package.json
{
  "type": "module"
}

// Or use .mjs extension
// app.mjs

Issue: Tables not rendering

Solution: Install and use remark-gfm 
npm install remark-gfm
import remarkGfm from 'remark-gfm'

<Markdown remarkPlugins={[remarkGfm]}>...</Markdown>

Issue: Styles not applying

Solutions:
  • Add className prop to wrapper
  • Use custom components with inline styles
  • Use CSS modules or styled-components

Issue: HTML tags showing as text

Solution: Use rehype-raw to parse HTML 
import rehypeRaw from 'rehype-raw'

<Markdown rehypePlugins={[rehypeRaw]}>...</Markdown>
