🎯 Multi-Element Capture Feature Plan

Core Concept: Command-Click Multi-Selection

User Flow: Press hotkey (Cmd+Shift+E or Ctrl+Shift+E) → Command/Ctrl-click multiple elements → Confirm selection → Capture all as group

  • Selection Mode: Hold Cmd/Ctrl while clicking to add elements to selection set
  • Visual Feedback: Each selected element gets numbered badge + persistent red outline
  • Action Panel: Floating panel shows count and actions (Capture, Chat, Clear)
  • Result: Creates single card with all elements combined OR multiple linked cards (user choice)

🎨 User Experience Flow

1. Activation User presses Cmd+Shift+E (Stash) or Ctrl+Shift+E (Canvas)
→ ElementSelector mounts in Shadow DOM
→ Mode banner appears (CANVAS/STASH)
→ Tooltip shows: "Click to select, Cmd+Click for multiple"
2. First Element Selection User clicks first element
→ Red outline + badge "1" appears
→ Action panel slides in (bottom-right)
→ Shows: "1 element selected"
3. Additional Selections User Cmd/Ctrl-clicks more elements
→ Each gets numbered badge (2, 3, 4...)
→ Action panel updates count
→ Click selected element again to deselect
4. Finalization User clicks "Capture" in action panel
→ Modal appears: "Combine as one card or create separate linked cards?"
→ User chooses merge strategy
→ Elements processed and saved

Data Structure Changes

New Type: MultiElementSelection

interface MultiElementSelection {
  elements: SelectedElement[];
  mode: 'canvas' | 'stash';
  timestamp: number;
}

interface SelectedElement {
  element: HTMLElement;
  index: number;
  selector: string;
  html: string;
  position: { x: number; y: number };
  dimensions: { width: number; height: number };
}

const handleClick = (e: MouseEvent) => {
  if (e.metaKey || e.ctrlKey) {
    // Multi-select mode
    toggleElementSelection(e.target);
  } else {
    // Single select (legacy)
    captureElement(e.target);
  }
};

Selection Management Functions

function toggleElementSelection(el) {
  const existing = selections
    .findIndex(s => s.element === el);

  if (existing >= 0) {
    // Deselect
    removeSelection(existing);
  } else {
    // Add to selection
    addSelection(el);
  }
}

function addSelection(el) {
  const selection = {
    element: el,
    index: selections.length + 1,
    selector: getSelector(el),
    html: sanitizeHTML(el.outerHTML),
    // ... metadata
  };
  setSelections([...selections, selection]);
}

Merge Strategies

Strategy Description Result Use Case
Combined Single Card Merge all elements into one HTML block 1 card with combined content Related content (e.g., product + reviews)

Hierarchical Parent-Child First element = parent, others = children 1 parent + N children with arrows Main article + supporting sections
Sequential Chain Elements connected in selection order N cards with sequential arrows (1→2→3) Step-by-step tutorials, timelines

▶ Chat Integration for Multi-Selection

Inline Chat with Multiple Elements

Workflow: User selects multiple elements → clicks "Chat with Selection" → inline chat opens with combined context

  • Context Building: All selected elements combined into single context string
  • System Prompt: "You are analyzing 3 elements from [domain]. Element 1: [html], Element 2: [html], Element 3: [html]"
  • Chat Window: Shows element count badge (e.g., "💬 Chat (3 elements)")
  • Save to Canvas: Creates single note card with conversation + all element references

Context Formatting for Claude

const buildMultiElementContext = (selections: SelectedElement[]) => {
  const context = selections.map((sel, idx) => `
    <element index="${idx + 1}">
      <selector>${sel.selector}</selector>
      <content>${sel.html}</content>
      <metadata>
        Position: ${sel.position.x}, ${sel.position.y}
        Size: ${sel.dimensions.width}x${sel.dimensions.height}
      </metadata>
    </element>
  `).join('\n');

  return `You are analyzing ${selections.length} elements from the page:\n${context}`;
};

⚠️ Edge Cases & Considerations

Nested Element Selection

Problem: User selects parent, then clicks child

Solution: Allow both, but show warning: "Element 2 is inside Element 1"

Element Limit

Problem: User selects 100+ elements

Solution: Set max 20 elements, show toast: "Maximum 20 elements"

Dynamic Content

Problem: Selected element disappears (SPA navigation)

Solution: Capture HTML immediately on selection, not on final capture

Large Combined Card

Problem: 10 elements = 500KB HTML

Solution: Warn if combined size > 200KB, suggest separate cards
