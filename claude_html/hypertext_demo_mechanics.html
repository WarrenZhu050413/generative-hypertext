Event Hooks

From my own experience, the demo binds selectionchange to track whether text is selected inside #article. When the selection collapses and the palette is not pinned, the overlay hides.

document.addEventListener('selectionchange', () => { const sel = document.getSelection(); if ((!sel || sel.isCollapsed) && !palettePinned) hidePalette(); });

Range Capture

Looking at this differently, the hotkey handler clones the active range so later DOM edits (the pill insertion) do not lose the original selection.

const range = sel.getRangeAt(0); activeRange = range.cloneRange(); selectedTextEl.textContent = `‚Äú${sel.toString()}‚Äù`;

Positioning

The problem is palette placement; I use the selection bounding rect and offset with window.scrollX/Y so the overlay appears just below the highlight.

palette.style.left = `${Math.max(rect.left + window.scrollX - 10, 12)}px`; palette.style.top = `${rect.bottom + window.scrollY + 12}px`;

Input Flow

I am curious when the instructions matter, so the input remembers the last value. Pressing Enter calls generateButton.click() to commit without leaving the keyboard.

Hotkey — Capture range, pin palette, preload instructions.
User types — Optional guidance for Claude.
Enter / Click — Palette hides, pill inserts, async resolution starts.

Insertion

From my own experience, replacing the selection with a span keeps React reconciliation simple. The span stores identifiers and the original text through dataset.

const wrapper = document.createElement('span'); wrapper.className = 'hyper-pill loading'; wrapper.dataset.id = `hyper-${++hyperlinkCounter}`; activeRange.deleteContents(); activeRange.insertNode(wrapper);

Prebaked Payload

I ran Claude manually with claude --dangerously-skip-permissions -p to fetch a JSON summary and reference for ‚Äúgradient descent‚Äù. The response populates PREBAKED_RESULTS, so the demo shows realistic output instantly.

Dynamic Choice

The LLM decides whether to surface the summary or the link. Instructions containing words like ‚Äúlink‚Äù bias toward URLs; otherwise there is a random 45% chance to present the reference when Claude already supplied one.
