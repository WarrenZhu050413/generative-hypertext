▶🔄 Complete Data Flow: Element Capture Pipeline
1. User Trigger
User presses Cmd+Shift+E (Canvas mode) or Alt+Shift+E (Stash mode)
2. Background Worker
Keyboard listener in background/index.ts intercepts command
Sends message: chrome.tabs.sendMessage({type: 'ACTIVATE_SELECTOR', mode: 'canvas'})
3. Content Script Receives
content/index.tsx listens via chrome.runtime.onMessage
Mounts ElementSelector component into Shadow DOM
4. User Interaction
ElementSelector highlights elements on hover
User clicks element → Capture initiated
5. Capture Pipeline
• Extract HTML from selected element
• Sanitize with DOMPurify
• Extract metadata (URL, title, domain, favicon)
• Compute relevant styles
• Generate unique ID: generateId() (timestamp + random)
6. Save to Storage
cardService.createCard(cardData)
Saves to chrome.storage.local under key 'cards'
Sets card.stashed = true/false based on mode
7. Broadcast Update
Dual event system:
• window.dispatchEvent('nabokov:cards-updated') (local)
• chrome.runtime.sendMessage({type: 'CARD_CREATED'}) (cross-context)
8. UI Auto-Refresh
Canvas/Side Panel listeners detect event
Re-fetch cards from storage
React state updates → UI re-renders
No page reload required!

▶🔌 Cross-Context Communication System

Dual Event System for Real-Time Synchronization

Local Events (Same Context)

// Broadcast
window.dispatchEvent(
  new CustomEvent('nabokov:cards-updated')
);

// Listen
window.addEventListener(
  'nabokov:cards-updated',
  handleUpdate
);

Use: Canvas → Canvas, Side Panel → Side Panel

Runtime Messages (Cross-Context)

// Broadcast
chrome.runtime.sendMessage({
  type: 'CARD_CREATED',
  cardId: 'abc123'
});

// Listen
chrome.runtime.onMessage.addListener(
  (msg, sender, sendResponse) => {...}
);

Use: Background ↔ Content ↔ Canvas ↔ Side Panel

Message Flow Examples

Message Type From To Payload Effect
ACTIVATE_SELECTOR Background Content Script {mode: 'canvas'|'stash'} Mount ElementSelector
OPEN_INLINE_CHAT Background Content Script {} Toggle InlineChatWindow
CARD_CREATED cardService All contexts {cardId, type} Refresh card lists
CARD_UPDATED cardService All contexts {cardId} Re-fetch and re-render
CARD_DELETED cardService All contexts {cardId} Remove from UI
Critical Pattern: All card operations in cardService.ts broadcast via BOTH local events AND runtime messages to ensure Canvas and Side Panel stay synchronized without page reloads.

Cross-Context Synchronization Example

// In cardService.ts
export async function updateCard(cardId: string, updates: Partial<Card>) {
  // 1. Update in storage
  await chrome.storage.local.set({ cards: updatedCards });

  // 2. Broadcast locally
  window.dispatchEvent(new CustomEvent('nabokov:cards-updated'));

  // 3. Broadcast cross-context
  chrome.runtime.sendMessage({
    type: 'CARD_UPDATED',
    cardId
  });
}

// In Canvas (useCanvasState.ts)
useEffect(() => {
  const handleUpdate = () => loadCards();
  window.addEventListener('nabokov:cards-updated', handleUpdate);
  chrome.runtime.onMessage.addListener(handleUpdate);
  return () => {
    window.removeEventListener('nabokov:cards-updated', handleUpdate);
  };
}, []);

Result: When Side Panel updates a card, Canvas auto-refreshes instantly without reload. Vice versa.

Stash Workflow

1. Card Stashed
User presses Alt+Shift+E → Captures element → Saves with stashed: true
2. Hidden from Canvas
Canvas filter: cards.filter(c => !c.stashed)
Card invisible on canvas but persists in storage
3. Visible in Side Panel
Side Panel filter: cards.filter(c => c.stashed === true)
Displays all stashed cards
4. User Actions
• Restore: Sets stashed: false → Appears on canvas
• Delete: Permanently removes from storage
• Edit: Update tags, content, metadata
5. Real-Time Sync
All actions broadcast 'nabokov:stash-updated' event
Canvas and Side Panel auto-refresh

🔑 Critical Implementation Patterns

State Refresh Pattern: NEVER use window.location.reload(). Always use event-based refresh via window.dispatchEvent('nabokov:cards-updated') for faster UX and no race conditions.
Shadow DOM Isolation: Content script components render in Shadow DOM to prevent CSS conflicts. Requires /** @jsxImportSource @emotion/react */ pragma.
Sanitization: ALWAYS sanitize HTML with DOMPurify before storing to chrome.storage.local, especially for contentEditable input and LLM responses.
Card Tags Safety: Always check card.tags && card.tags.length or card.tags?.length (tags may be undefined on old cards).
Dual Event System: All card operations broadcast via BOTH local events (window.dispatchEvent) AND runtime messages (chrome.runtime.sendMessage) for cross-context sync.
ID Generation: Use generateId() from @/utils/storage for all IDs (timestamp + random).
