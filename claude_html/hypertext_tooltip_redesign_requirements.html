Hypertext Tooltip Redesign — UI Requirements & Implementation

Core UI Requirements

  • Compact Footprint: Tooltip maintains ≈300px width to avoid obscuring page content
  • Message Display: Shows only the most recent assistant message and most recent user prompt — no history scrolling
  • Typography Consistency: Prompt textarea uses identical font-size and line-height as chat output text
  • Text Size Toggle: Single button in tooltip header toggles between normal and large text sizes for both prompt and output
  • Primary Action: Clicking the tooltip body opens a dedicated chat page in a new window/tab
  • Persistence Mechanism: Dedicated chat page loads conversation state from sessionStorage payload saved when tooltip is clicked
  • Visual Identity: Nabokov-inspired palette with deep red accents (#8B0000), warm paper background (#F5F5DC), and gold highlights
  • Accessibility: All controls include aria-label attributes, button states, keyboard support (Enter to send, Esc to dismiss)

Implementation Notes

1. Tooltip Component Structure

  • Create header with text-size toggle button (aria-label: "Toggle text size")
  • Render most recent assistant message in scrollable output area
  • Render textarea for user prompt with matching typography
  • Add click handler to entire tooltip that saves state to sessionStorage and opens chat page
  • Implement Esc key listener to dismiss tooltip

2. Text Size State Management

  • Define two font-size presets (e.g., 14px normal, 18px large)
  • Store text-size preference in localStorage or component state
  • Apply font-size CSS variable to both output area and textarea
  • Ensure line-height scales proportionally (e.g., 1.5 ratio)

3. Dedicated Chat Page

  • Create standalone HTML page (chat.html) with full conversation UI
  • On tooltip click, serialize conversation context to sessionStorage with unique key
  • Chat page reads sessionStorage on load to restore conversation history
  • Implement full chat features (message history, scroll, image upload) on dedicated page

4. Nabokov Palette Application

  • Define CSS custom properties: --nabokov-red: #8B0000, --nabokov-gold: #FFD700, --paper-bg: #F5F5DC
  • Apply warm background gradient to tooltip body
  • Use deep red for header, borders, and toggle button accents
  • Apply gold highlights to hover states and active elements

5. Accessibility & Keyboard Support

  • Add role="dialog" and aria-modal="true" to tooltip container
  • Label toggle button with aria-label="Toggle between normal and large text size"
  • Implement Enter key in textarea to submit message (with Shift+Enter for newline)
  • Bind Esc key to dismiss tooltip and return focus to trigger element
  • Ensure focus trap within tooltip when open, restore focus on close
  • Use aria-live="polite" for assistant message updates
Key Engineering Constraint: The tooltip must remain stateless regarding conversation history — all message persistence happens via sessionStorage handoff to the dedicated chat page. This keeps the tooltip lightweight and avoids DOM bloat from long conversations.
