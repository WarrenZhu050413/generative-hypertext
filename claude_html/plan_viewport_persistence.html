‚ö†Ô∏è Potential Issues & Mitigations ‚ñ∂

Issue 1: fitView Conflicts

Problem: fitView prop overrides defaultViewport
Solution: Remove fitView prop, use programmatic fitView() only for first-time users

Issue 2: Debounce Timing

Problem: User might refresh before save completes (2s debounce)
Solution: ‚Ä¢ Save on beforeunload event (no debounce)
‚Ä¢ Or reduce debounce to 500ms for viewport (faster than card saves)

Issue 3: Very Large Canvases

Problem: Viewport might be way outside card bounds
Solution: Validate viewport is reasonable on load, fall back to fitView if needed

Issue 4: Different Screen Sizes

Problem: Saved viewport from desktop might look odd on mobile
Solution: Accept this limitation (canvas is primarily desktop tool) or detect screen size changes

Issue 5: Storage Race Conditions

Problem: Multiple saves happening simultaneously
Solution: Use single debounced save ref, clear timeout before new save
