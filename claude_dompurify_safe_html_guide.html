DOMPurify-Safe HTML: Complete Guide for LLM Prompts

⚠️ Critical Question Answered

Will re-sanitization break the AI-generated HTML?

Answer: It CAN, but ONLY if the LLM generates unsafe HTML. The solution is to give the LLM explicit rules about what HTML/CSS is safe.

This guide provides the exact constraints to include in your LLM prompts to ensure the generated HTML passes through DOMPurify unchanged.

How DOMPurify Works

▶ What DOMPurify Removes

DOMPurify parses HTML and removes elements/attributes that could execute JavaScript or load external malicious content.

Removed by Default

❌ Dangerous Tags

  • <script>
  • <iframe>
  • <object>
  • <embed>
  • <applet>
  • <link> (can load CSS)
  • <base> (URL hijacking)

❌ Event Handler Attributes

  • onclick, onload
  • onerror, onmouseover
  • All on* attributes
  • javascript: in href
  • data: URLs (configurable)

Configuration-Dependent Removal

What gets removed depends on your DOMPurify config:

Your Current Config (from code)
DOMPurify.sanitize(html, {
  ALLOWED_TAGS: [
    'div', 'span', 'p', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6',
    'ul', 'ol', 'li', 'a', 'img', 'code', 'pre', 'blockquote',
    'table', 'thead', 'tbody', 'tr', 'th', 'td', 'style',
    'br', 'strong', 'em', 'section', 'article'
  ],
  ALLOWED_ATTR: ['style', 'class', 'id', 'href', 'src', 'alt'],
  ALLOW_DATA_ATTR: false,  // Blocks data-* attributes
});

Key Insight: Any tag NOT in ALLOWED_TAGS is removed. Any attribute NOT in ALLOWED_ATTR is removed.

Safe HTML Constraints for LLM

▶ Complete Safe HTML Ruleset

Include these constraints in your LLM system prompt to ensure DOMPurify-safe output:

System Prompt Addition: DOMPurify Safety Rules
CRITICAL: Your HTML will be sanitized for security. Follow these rules EXACTLY:

ALLOWED HTML TAGS (use ONLY these):
- Structure: div, span, section, article, header, footer
- Headings: h1, h2, h3, h4, h5, h6
- Text: p, br, strong, em, blockquote
- Lists: ul, ol, li
- Links/Media: a, img
- Code: code, pre
- Tables: table, thead, tbody, tr, th, td
- Styling: style (for CSS)

ALLOWED ATTRIBUTES (use ONLY these):
- style (inline CSS)
- class (CSS class names)
- id (element IDs)
- href (for <a> tags - HTTPS URLs only, NO javascript:)
- src (for <img> tags - HTTPS URLs only, NO data: URLs)
- alt (for <img> tags)

FORBIDDEN (will be REMOVED):
❌ Tags: script, iframe, object, embed, svg, canvas, form, input, button
❌ Attributes: onclick, onload, onerror, on* (any event handlers)
❌ Attributes: data-*, aria-*, type, name, value
❌ URLs: javascript:, data:, vbscript:
❌ CSS: expression(), -moz-binding, behavior:

SAFE CSS PROPERTIES (use these freely):
✅ Layout: display, position, top, left, width, height, margin, padding
✅ Flexbox/Grid: flex, grid, justify-content, align-items, gap
✅ Typography: font-family, font-size, font-weight, line-height, color
✅ Colors: color, background, background-color, border-color
✅ Borders: border, border-radius, box-shadow
✅ Gradients: linear-gradient(), radial-gradient()
✅ Transforms: transform, transition

OUTPUT FORMAT (use this structure):
<style>
  /* Your safe CSS here */
  .container { ... }
  .heading { ... }
</style>
<div class="container">
  <!-- Your safe HTML here -->
</div>

Testing Strategy

▶ How to Test if HTML Survives DOMPurify

Test Utility Function

src/utils/test-sanitization.ts
import DOMPurify from 'dompurify';

/**
 * Test if HTML survives DOMPurify sanitization unchanged
 * (or with only expected changes like whitespace normalization)
 */
export function testSanitization(html: string): {
  passed: boolean;
  before: string;
  after: string;
  diff?: string;
} {
  const sanitized = DOMPurify.sanitize(html, {
    ALLOWED_TAGS: [
      'div', 'span', 'section', 'article', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6',
      'p', 'br', 'strong', 'em', 'ul', 'ol', 'li', 'a', 'img', 'code', 'pre',
      'blockquote', 'table', 'thead', 'tbody', 'tr', 'th', 'td', 'style'
    ],
    ALLOWED_ATTR: ['style', 'class', 'id', 'href', 'src', 'alt'],
    ALLOW_DATA_ATTR: false,
  });

  // Normalize whitespace for comparison
  const normalizedBefore = html.replace(/\s+/g, ' ').trim();
  const normalizedAfter = sanitized.replace(/\s+/g, ' ').trim();

  const passed = normalizedBefore === normalizedAfter;

  return {
    passed,
    before: html,
    after: sanitized,
    diff: !passed ? `Length before: ${html.length}, after: ${sanitized.length}` : undefined,
  };
}

// Usage in tests
const llmOutput = `
<style>
  .container { background: #F5F5DC; }
</style>
<div class="container"><h1>Title</h1></div>
`;

const result = testSanitization(llmOutput);
console.log(result.passed ? '✅ Safe' : '❌ Will be modified');
if (!result.passed) {
  console.log('Before:', result.before);
  console.log('After:', result.after);
}

Handling Edge Cases

▶ What If LLM Generates Forbidden Content?

Despite clear instructions, LLMs sometimes generate forbidden content. Here's how to handle it:

Strategy 1: Detect & Warn User

Post-processing Check
export async function beautifyWithValidation(
  card: Card,
  mode: 'recreate' | 'organize'
): Promise<string> {
  // Get LLM output
  const llmOutput = mode === 'recreate'
    ? await recreateDesign(card, screenshot)
    : await organizeInformation(card);

  // Sanitize
  const sanitized = DOMPurify.sanitize(llmOutput, { /* config */ });

  // Check if substantial content was removed
  const sizeBefore = llmOutput.length;
  const sizeAfter = sanitized.length;
  const lossPercentage = ((sizeBefore - sizeAfter) / sizeBefore) * 100;

  if (lossPercentage > 10) {
    console.warn(
      `[Beautification] ${lossPercentage.toFixed(1)}% of content removed by sanitization`,
      '\nBefore length:', sizeBefore,
      '\nAfter length:', sizeAfter
    );

    // Optional: Show warning to user
    alert(`Warning: Some content was removed for security. The result may not match expectations.`);
  }

  return sanitized;
}

Strategy 2: Retry with Stricter Prompt

Retry Logic
async function beautifyWithRetry(
  card: Card,
  maxRetries: number = 2
): Promise<string> {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    const llmOutput = await organizeInformation(card);
    const sanitized = DOMPurify.sanitize(llmOutput, { /* config */ });

    const lossPercentage =
      ((llmOutput.length - sanitized.length) / llmOutput.length) * 100;

    if (lossPercentage < 5) {
      // Acceptable loss (whitespace, etc.)
      return sanitized;
    }

    console.warn(
      `Attempt ${attempt + 1}: ${lossPercentage.toFixed(1)}% content removed. Retrying...`
    );

    // Add stricter instructions for retry
    // (modify prompt to emphasize safety rules)
  }

  // Final attempt failed, return best result
  return sanitized;
}

Strategy 3: Fallback to Original

Safe Fallback
async function beautifyWithFallback(card: Card): Promise<string> {
  try {
    const beautified = await organizeInformation(card);
    const sanitized = DOMPurify.sanitize(beautified, { /* config */ });

    // If result is too short (major content loss), fallback
    if (sanitized.length < beautified.length * 0.8) {
      console.warn('Too much content removed, using original');
      return card.content; // Original sanitized HTML
    }

    return sanitized;
  } catch (error) {
    console.error('Beautification failed:', error);
    return card.content; // Fallback to original
  }
}

Final Recommendations

Best Practices

  1. Always include safety constraints in system prompts - The LLM needs explicit rules about what's forbidden
  2. Test with edge cases before deployment - Use the test utility to validate output
  3. Monitor sanitization loss - Log when >10% of content is removed
  4. Provide user feedback - If content was removed, let user know
  5. Keep original HTML as fallback - Always preserve card.content
  6. Consider allowlist expansion carefully - If you need SVG, add it to ALLOWED_TAGS explicitly

What NOT to Do

  • ❌ Don't skip re-sanitization (LLM output can't be trusted)
  • ❌ Don't assume LLM will follow rules perfectly (it won't)
  • ❌ Don't add event handlers to ALLOWED_ATTR (massive security hole)
  • ❌ Don't disable DOMPurify to "fix" broken output
  • ❌ Don't use ALLOW_UNKNOWN_PROTOCOLS (enables javascript: URLs)
