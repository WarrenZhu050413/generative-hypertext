Data Structure - Technical Definition

Core Definition

A data structure is a specialized format for organizing, processing, retrieving, and storing data in a computer's memory, enabling efficient access and modification.

More formally: A data structure is a collection of data values, the relationships among them, and the operations that can be applied to the data.

Formal Mathematical Definition:
DS = (D, R, O)

  • Constraints: Rules governing structure and access

â–¶Abstract vs Concrete Data Structures

Abstract Data Type (ADT)

Definition: Mathematical model defining data type by its behavior (operations) from user's perspective, independent of implementation.

Examples:

  • List (ordered collection)
  • Stack (LIFO)
  • Queue (FIFO)
  • Set (unique elements)
  • Map/Dictionary (key-value pairs)
  • Tree (hierarchical structure)
  • Graph (nodes and edges)

Concrete Data Structure

Definition: Actual implementation of an ADT in memory, specifying how data is stored and operations are performed.

Examples:

  • Array (contiguous memory)
  • Linked List (node pointers)
  • Hash Table (array + hash function)
  • Binary Search Tree (pointer-based)
  • Heap (array-based tree)
  • B-Tree (disk-optimized tree)

Key Distinction

ADT = WHAT (interface, behavior, operations) â€” "A stack supports push and pop"

Concrete DS = HOW (implementation, memory layout) â€” "A stack implemented using an array"

Example: The Stack ADT can be implemented using Array, Linked List, or Dynamic Array concrete structures.

â–¶Common Operations & Complexity

Standard Operations

Access Operations

  • Access(i) - Retrieve element at position i
  • Search(x) - Find element x
  • Traverse() - Visit all elements

Modification Operations

  • Insert(x, i) - Add x at position i
  • Delete(i) - Remove element at position i
  • Update(i, x) - Modify element at i

Query Operations

  • Size() - Number of elements
  • IsEmpty() - Check if empty
  • Contains(x) - Check membership

Iterator Operations

  • First() - Get first element
  • Next() - Move to next
  • HasNext() - Check if more elements

â–¶Advanced Concepts

Persistent Data Structures

Definition: Data structures that preserve previous versions when modified, enabling access to historical states.

Types:

  • Partially Persistent: All versions readable, only newest writable
  • Fully Persistent: All versions readable and writable
  • Confluent Persistent: Versions can be merged

Immutable Data Structures

Definition: Structures that cannot be modified after creation; operations return new structures.

Benefits: Thread-safety, easier reasoning, cacheable, undo/redo support

Implementation: Structural sharing to minimize copying

Concurrent Data Structures

Definition: Structures designed for safe concurrent access by multiple threads.

Techniques: Lock-free algorithms, Compare-and-Swap (CAS), Memory barriers

Examples: Concurrent HashMap, Lock-free Queue, Skip List

Cache-Oblivious Data Structures

Definition: Structures optimized for cache performance without knowing cache parameters.

Principle: Recursive decomposition achieves optimal cache usage at all memory hierarchy levels

Examples: Cache-oblivious B-tree, van Emde Boas layout

Examples:

Summary: Data Structure Essence

A data structure is fundamentally about organizing information to solve problems efficiently. The "right" structure depends on:

  • What operations you need (access, search, insert, delete)
  • How often you perform each operation
  • Whether you know the data size in advance
  • Memory constraints and locality
  • Whether data needs ordering
  • Concurrency requirements
  • Cache/disk considerations
  • Acceptable trade-offs (time vs space)

The art of programming is choosing the right data structure for the problem at hand.
