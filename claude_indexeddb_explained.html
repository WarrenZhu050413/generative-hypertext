üì¶ IndexedDB: Complete Guide

What is IndexedDB?
A low-level API for client-side storage of significant amounts of structured data (including files/blobs). Think of it as a NoSQL database built into your browser that persists data locally and works offline.

üéØ Core Concepts (Most Important First)

The 3 Essential Components

1. Database

  • Container for everything
  • Per-origin (domain-specific)
  • Has a name and version number
  • Example: "nabokov-cards"

2. Object Store

  • Like a SQL table
  • Stores JavaScript objects
  • Each has a key path or auto-increment
  • Example: "cards" store

3. Index

  • Fast lookup by specific field
  • Like SQL indexes - speeds up queries
  • Example: Index on metadata.created for date sorting

üíª Basic Usage: Step-by-Step

‚ñ∂Step 1: Open/Create Database 
// Request to open database (creates if doesn't exist)
const request = indexedDB.open('nabokov-cards', 1);
//                               ‚Üë name         ‚Üë version

// This is ASYNC - need event handlers
request.onerror = (event) => {
  console.error('Database error:', event.target.error);
};

request.onsuccess = (event) => {
  const db = event.target.result;
  console.log('Database opened:', db);
};

// Called ONLY when database is created or version changes
request.onupgradeneeded = (event) => {
  const db = event.target.result;

  // Create object store if it doesn't exist
  if (!db.objectStoreNames.contains('cards')) {
    const objectStore = db.createObjectStore('cards', { keyPath: 'id' });
    //                                                    ‚Üë primary key field

    // Create indexes for fast lookups
    objectStore.createIndex('by-created', 'metadata.created', { unique: false });
    objectStore.createIndex('by-url', 'source.url', { unique: false });
  }
};
Key Points:
  • Version number: Increment to trigger schema changes
  • onupgradeneeded: Only place you can modify schema (add stores/indexes)
  • keyPath: Field that acts as primary key (must be unique)
  • Indexes: Optional but make queries much faster
‚ñ∂Step 2: Transactions & CRUD Operations

Why Transactions?

All database operations happen in transactions for data integrity. Think: "atomic operations that either fully succeed or fully fail."

CREATE (Add)

// Start a read-write transaction
const tx = db.transaction(['cards'], 'readwrite');
const store = tx.objectStore('cards');

const card = {
  id: 'uuid-123',
  element: { html: '...' },
  conversation: [],
  // ... rest of card data
};

// Add to store
const request = store.add(card);

request.onsuccess = () => {
  console.log('Card added!');
};

request.onerror = () => {
  console.error('Failed to add');
};

READ (Get)

// Read-only transaction (faster)
const tx = db.transaction(['cards'], 'readonly');
const store = tx.objectStore('cards');

// Get single item by key
const request = store.get('uuid-123');

request.onsuccess = (event) => {
  const card = event.target.result;
  console.log('Found card:', card);
};

// Get ALL items
const getAllRequest = store.getAll();
getAllRequest.onsuccess = (event) => {
  const allCards = event.target.result;
  console.log('All cards:', allCards);
};

UPDATE (Put)

const tx = db.transaction(['cards'], 'readwrite');
const store = tx.objectStore('cards');

// Get existing card
const getRequest = store.get('uuid-123');
getRequest.onsuccess = (event) => {
  const card = event.target.result;

  // Modify it
  card.conversation.push({
    role: 'user',
    content: 'New message'
  });

  // Put back (overwrites)
  const putRequest = store.put(card);
  putRequest.onsuccess = () => {
    console.log('Card updated!');
  };
};

DELETE

const tx = db.transaction(['cards'], 'readwrite');
const store = tx.objectStore('cards');

const request = store.delete('uuid-123');

request.onsuccess = () => {
  console.log('Card deleted!');
};

// Delete ALL items in store
const clearRequest = store.clear();
clearRequest.onsuccess = () => {
  console.log('All cards deleted!');
};
‚ñ∂Step 3: Querying with Indexes & Cursors

Using Indexes for Fast Queries

const tx = db.transaction(['cards'], 'readonly');
const store = tx.objectStore('cards');

// Get index
const index = store.index('by-created');

// Query by index (get all cards sorted by creation date)
const request = index.getAll();
request.onsuccess = (event) => {
  const sortedCards = event.target.result;
  console.log('Cards by date:', sortedCards);
};

// Get cards from specific URL
const urlIndex = store.index('by-url');
const urlRequest = urlIndex.getAll('https://example.com');
urlRequest.onsuccess = (event) => {
  const cardsFromUrl = event.target.result;
};

Using Cursors (Advanced Iteration)

// Cursor lets you iterate through results one-by-one
const tx = db.transaction(['cards'], 'readonly');
const store = tx.objectStore('cards');

const request = store.openCursor();

request.onsuccess = (event) => {
  const cursor = event.target.result;

  if (cursor) {
    console.log('Card:', cursor.key, cursor.value);

    // Move to next item
    cursor.continue();
  } else {
    console.log('Done iterating');
  }
};

üéÅ Modern Wrapper: idb Library

Why use idb? Native IndexedDB is callback-based and verbose. The idb library wraps it with Promises/async-await for cleaner code.

‚ùå Native IndexedDB (Ugly)

const request = indexedDB.open('db', 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  db.createObjectStore('cards', { keyPath: 'id' });
};

request.onsuccess = (event) => {
  const db = event.target.result;
  const tx = db.transaction(['cards'], 'readwrite');
  const store = tx.objectStore('cards');
  const addRequest = store.add(card);

  addRequest.onsuccess = () => {
    console.log('Added!');
  };
};

‚úÖ With idb Library (Clean)

import { openDB } from 'idb';

const db = await openDB('db', 1, {
  upgrade(db) {
    db.createObjectStore('cards', { keyPath: 'id' });
  },
});

// Add card
await db.add('cards', card);
console.log('Added!');

// Get card
const card = await db.get('cards', 'uuid-123');

// Get all
const allCards = await db.getAll('cards');

üèóÔ∏è Complete Example: Nabokov Card Database

‚ñ∂Full Implementation with idb 
// db.ts
import { openDB, DBSchema, IDBPDatabase } from 'idb';

// Define TypeScript schema
interface NabokovDB extends DBSchema {
  cards: {
    key: string;
    value: ClippedCard;
    indexes: {
      'by-created': number;
      'by-url': string;
    };
  };
}

class CardDatabase {
  private db: IDBPDatabase | null = null;

  async init() {
    this.db = await openDB('nabokov-cards', 1, {
      upgrade(db) {
        const store = db.createObjectStore('cards', { keyPath: 'id' });
        store.createIndex('by-created', 'metadata.created');
        store.createIndex('by-url', 'source.url');
      },
    });
  }

  async addCard(card: ClippedCard): Promise {
    return this.db!.add('cards', card);
  }

  async getCard(id: string): Promise {
    return this.db!.get('cards', id);
  }

  async getAllCards(): Promise {
    return this.db!.getAll('cards');
  }

  async getCardsByUrl(url: string): Promise {
    return this.db!.getAllFromIndex('cards', 'by-url', url);
  }

  async getCardsSortedByDate(): Promise {
    return this.db!.getAllFromIndex('cards', 'by-created');
  }

  async updateCard(id: string, updates: Partial): Promise {
    const card = await this.getCard(id);
    if (!card) throw new Error('Card not found');

    const updated = { ...card, ...updates };
    return this.db!.put('cards', updated);
  }

  async deleteCard(id: string): Promise {
    return this.db!.delete('cards', id);
  }

  async searchCards(query: string): Promise {
    const allCards = await this.getAllCards();
    return allCards.filter(card =>
      card.element.html.toLowerCase().includes(query.toLowerCase()) ||
      card.source.title.toLowerCase().includes(query.toLowerCase())
    );
  }
}

export const cardDB = new CardDatabase();

// Usage in app
await cardDB.init();

// Add a card

  id: crypto.randomUUID(),

', /* ... */ },

});

// Get all cards

‚ö° Performance Considerations

DO Batch Operations

// Good: Single transaction
const tx = db.transaction(['cards'], 'readwrite');
const store = tx.objectStore('cards');

await store.add(card1);
await store.add(card2);
await store.add(card3);

await tx.done;

DON'T Multiple Transactions

// Bad: 3 separate transactions
await db.add('cards', card1);
await db.add('cards', card2);
await db.add('cards', card3);

// Much slower!

DO Use Indexes

// Fast: Index lookup
const cards = await db
  .getAllFromIndex(
    'cards',
    'by-url',
    url
  );

Storage Limits & Quotas

  • Chrome: Up to 80% of disk space (temporary) or unlimited (persistent)
  • Firefox: Up to 50% of free disk space (temporary) or 2GB (persistent)
  • Safari: ~1GB limit
  • Check quota: navigator.storage.estimate()
  • Request persistent: navigator.storage.persist()

üîß Common Patterns

‚ñ∂Pattern: Pagination 
async function getCardsPaginated(page: number, pageSize: number) {
  const tx = db.transaction(['cards'], 'readonly');
  const store = tx.objectStore('cards');
  const index = store.index('by-created');

  let count = 0;
  let results: ClippedCard[] = [];
  const start = page * pageSize;
  const end = start + pageSize;

  const cursor = await index.openCursor(null, 'prev'); // Newest first

  for await (const cursor of cursorIterate(cursor)) {
    if (count >= start && count < end) {
      results.push(cursor.value);
    }
    count++;
    if (count >= end) break;
  }

  return results;
}

async function saveCard(card: ClippedCard) {
  // 1. Save to IndexedDB immediately
  await cardDB.addCard(card);

  // 2. Try to sync to server
  try {

  }
}

‚ñ∂Pattern: Migration (Version Upgrades) 
const db = await openDB('nabokov-cards', 2, {
  upgrade(db, oldVersion, newVersion, transaction) {
    // Version 0 ‚Üí 1: Initial schema
    if (oldVersion < 1) {
      const store = db.createObjectStore('cards', { keyPath: 'id' });
      store.createIndex('by-created', 'metadata.created');
    }

    // Version 1 ‚Üí 2: Add tags field
    if (oldVersion < 2) {
      const store = transaction.objectStore('cards');
      store.createIndex('by-tag', 'metadata.tags', { multiEntry: true });

      // Migrate existing data
      const cursor = await store.openCursor();
      for await (const cursor of cursorIterate(cursor)) {
        const card = cursor.value;
        card.metadata.tags = card.metadata.tags || [];
        cursor.update(card);
      }
    }
  },
});

‚ùå Common Mistakes

  • Forgetting transactions close: Always use await tx.done or wait for request completion
  • Schema changes outside onupgradeneeded: Can only modify schema during upgrade
  • Not handling errors: Always add error handlers or try-catch
  • Blocking the main thread: IndexedDB is async, but iterating huge datasets can still block
  • Storing non-cloneable objects: Can't store functions, DOM elements, or circular references

üéì Summary

When to use IndexedDB:

  • ‚úÖ Large amounts of data (images, conversations, documents)
  • ‚úÖ Structured data (objects with relationships)
  • ‚úÖ Offline-first apps (PWAs, web apps that work without internet)
  • ‚úÖ Complex queries (need indexes, sorting, filtering)

When NOT to use IndexedDB:

  • ‚ùå Small settings/config (use localStorage)
  • ‚ùå Simple key-value pairs (localStorage is simpler)
  • ‚ùå Server-authoritative data (fetch from API instead)
